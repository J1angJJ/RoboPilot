# Troubleshooting

This page covers common RoboPilot setup and migration workflow issues. RoboPilot does not require ROS, ROS2, `catkin_make`, `colcon`, launch execution, generated node execution, or robot hardware for its default workflows.

## `robopilot` Command Not Found

Install RoboPilot in the Python environment you are using:

```bash
pip install robopilot
```

For a source checkout:

```bash
python -m pip install -e ".[dev]"
```

Then verify:

```bash
robopilot --help
```

If multiple Python installations are present, run:

```bash
python -m pip show robopilot
```

## Supported Python Versions

RoboPilot currently claims Python 3.10 and 3.11 support. Package metadata declares:

```txt
>=3.10,<3.12
```

Python 3.12 is not claimed yet. Python 3.13 is not supported yet.

## VSCode Cannot Find RoboPilot

VSCode may not inherit your shell or conda environment PATH.

Try one of these:

- Start VSCode from an activated environment.
- Set `robopilot.executablePath` to the full path of the `robopilot` executable.
- On Windows, verify the executable path from the same terminal profile VSCode uses.

For conda:

```powershell
conda activate robopilot
where robopilot
```

Use the printed path as `robopilot.executablePath` if needed.

## PowerShell Encoding or Box Drawing Mojibake

Some Windows PowerShell sessions display Rich table borders as mojibake when the console encoding is not UTF-8. This is usually display-only; command results and output files are still valid.

Options:

```powershell
$env:PYTHONIOENCODING='utf-8'
robopilot --help
```

or use Windows Terminal / PowerShell 7 with UTF-8 output.

For automation, prefer `--json` where available instead of parsing human-readable Rich output.

## Windows `.pytest_tmp` Access Denied

If pytest cannot access the default temp directory, use a workspace-local basetemp:

```bash
python -m pytest --basetemp=".pytest_tmp" -p no:cacheprovider
```

If a previous run left a locked temp directory, use a fresh name:

```bash
python -m pytest --basetemp=".pytest_tmp_v2" -p no:cacheprovider
```

Close editors or terminals that may be holding files open before deleting temp directories.

## Existing Scaffold Output Conflicts

`migrate-scaffold` refuses to overwrite existing files by default.

Use a fresh output directory:

```bash
robopilot migrate-scaffold --plan migration_plan.yaml --output .pytest_tmp_v115_manual/ros2_scaffold
```

Only use `--overwrite` after reviewing the target directory:

```bash
robopilot migrate-scaffold --plan migration_plan.yaml --output .pytest_tmp_v115_manual/ros2_scaffold --overwrite
```

RoboPilot overwrites only intended scaffold files, not arbitrary source project files.

## Missing Migration Plan

If validation or scaffold commands report that a migration plan cannot be loaded, confirm the path and regenerate the plan if needed:

```bash
robopilot migrate-plan --from examples/ros1_migration_demo --to ros2 --output .pytest_tmp_v115_manual/migration_plan.yaml
robopilot migrate-plan-validate --plan .pytest_tmp_v115_manual/migration_plan.yaml
```

## Invalid Migration Plan

Run validation before scaffold generation:

```bash
robopilot migrate-plan-validate --plan migration_plan.yaml
```

If required fields are missing or the target is unsupported, regenerate the plan with `robopilot migrate-plan`.

## Missing Scaffold Directory

If scaffold validation reports a missing scaffold path, generate the scaffold first:

```bash
robopilot migrate-scaffold --plan migration_plan.yaml --output .pytest_tmp_v115_manual/ros2_scaffold
robopilot migrate-scaffold-validate --plan migration_plan.yaml --scaffold .pytest_tmp_v115_manual/ros2_scaffold
```

## Missing `MIGRATION_NOTES.md`

`MIGRATION_NOTES.md` is required because it carries safety notes and manual migration reminders. Regenerate the scaffold or restore the file from the generated output.

## Placeholder Safety Warnings

Generated placeholder files should say they were generated by RoboPilot, are not automatic migration, require TODO/manual review, and mention QoS or launch review where relevant.

If a check fails, restore the placeholder safety wording or regenerate the scaffold.

## VSCode Missing `migration_plan.json`

Run:

```txt
RoboPilot: Generate Migration Plan
```

before:

```txt
RoboPilot: Preview Migration Scaffold
```

The extension stores the plan under `.robopilot_vscode/migration_plan.json` by default.

## VSCode Missing `.robopilot_vscode/ros2_scaffold`

Run:

```txt
RoboPilot: Generate Migration Scaffold
```

before:

```txt
RoboPilot: Validate Migration Scaffold
RoboPilot: Generate Scaffold Report
```

If the scaffold already exists and generation fails due to conflicts, review `.robopilot_vscode/ros2_scaffold` or change `robopilot.outputDirectory`.

## No ROS Required

RoboPilot's migration workflow is static. It can help plan, scaffold, validate, and report migration work, but it does not prove ROS2 runtime correctness.

After manual migration, use real ROS2 tooling in your own ROS2 environment when you are ready.
